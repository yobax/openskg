# OpenSKG

**Open - Security Knowledge Graph**

OpenSKG is an open-source project that structures and connects information security knowledge into a unified knowledge graph.

Rather than treating security as isolated frameworks, products, or documents, OpenSKG organizes knowledge through relationships between controls, assets, threats, risks, services, policies, procedures, operations, evidence, and compliance.

The goal is to create a reusable and extensible knowledge base that helps security professionals, engineers, auditors, researchers, and AI systems understand security from first principles.

## Vision

Information security is built upon the same fundamental concepts, regardless of frameworks or vendors.

OpenSKG focuses on those fundamentals and maps them to international standards, products, and operational practices.

Instead of learning every framework independently, learn the underlying security concepts once and understand how they relate everywhere.

## Scope

OpenSKG currently covers:

- Security Controls
- Assets
- Threats
- Risks
- Security Frameworks
- Compliance Standards
- Security Services
- Policies
- Procedures
- Operational Tasks
- Evidence
- Security Logs
- Assessments
- AI-ready Knowledge

## Design Principles

- Framework-independent
- Vendor-neutral
- Evidence-based
- AI-friendly
- Community-driven
- Open and extensible

## Why OpenSKG?

Security knowledge is often fragmented.

Controls appear in different standards with different names.
Products implement similar capabilities in different ways.
Operational knowledge is scattered across documentation.

OpenSKG connects these pieces into a single structured knowledge graph.

## License

This project is released as an open-source community project.

Contributions are welcome.

Together, we can build an open knowledge graph for information security.

# OpenSKG

**Open - Security Knowledge Graph**

OpenSKG は、情報セキュリティに関する知識を、一つの知識グラフ（Knowledge Graph）として整理・関連付けることを目的としたオープンソースプロジェクトです。

セキュリティを「認証制度」「製品」「手順」といった個別の情報として扱うのではなく、それらを相互に関連付け、一つの構造化された知識として管理します。

対象となる知識は、コントロール、資産、脅威、リスク、サービス、ポリシー、手順、運用、証跡、コンプライアンスなど多岐にわたります。

OpenSKG は、セキュリティ担当者、エンジニア、監査担当者、研究者、そしてAIが共通して利用できる知識基盤を目指しています。

## ビジョン

情報セキュリティの本質は、製品や認証制度によって変わるものではありません。

OpenSKG は、それらの共通となる基本概念を整理し、各種フレームワークや製品、運用方法との対応関係を構築します。

一つひとつの認証制度を個別に学ぶのではなく、本質となるセキュリティの考え方を理解し、それを様々な規格や製品へ応用できる世界を目指します。

## 対象

現在、以下の分野を対象としています。

- セキュリティコントロール
- 資産管理
- 脅威
- リスク
- セキュリティフレームワーク
- コンプライアンス
- セキュリティサービス
- ポリシー
- プロシージャ
- 運用
- 証跡
- ログ
- アセスメント
- AI向け知識構造

## 設計思想

- フレームワーク非依存
- ベンダー中立
- 証拠に基づく構造
- AIが利用しやすい設計
- コミュニティ主導
- 継続的な拡張を前提とする

## OpenSKG が目指すもの

現在のセキュリティ知識は、多くの文書や製品、規格に分散しています。

同じコントロールであっても規格によって名称が異なり、同じ機能でも製品によって実装方法が異なります。

OpenSKG は、それらを一つの構造として関連付け、再利用可能な知識グラフとして提供することを目指します。

## ライセンス

本プロジェクトはオープンソースとして公開されています。

皆様からの改善提案やコントリビューションを歓迎します。

情報セキュリティの知識を、共に育てていきましょう。
